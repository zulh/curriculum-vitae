# Live CV — Ahmad Zulhilmi Ghazali — Design Spec

**Date:** 2026-04-12  
**Status:** Approved

## Overview

A single-page, animated personal CV website for Ahmad Zulhilmi Ghazali, a .NET developer based in Kuala Lumpur. The site translates his existing PDF CV into an interactive, visually engaging web experience with scroll animations, a typing hero, and an animated timeline — all in his original navy & sky blue palette.

## Architecture

### Data Layer

All CV content lives in a single file: `src/data/cv.js`. This is the only file that needs to be edited when CV content changes. No section component imports it directly — `App.jsx` imports it once and passes slices down as props.

```js
export const cv = {
  personal: { name, title, phone, email, location },
  profile: String,
  experience: [
    {
      company: String,
      role: String,
      period: String,
      logo: String,          // path to logo in src/assets/logos/ (e.g. "expro.png", "petronas.png") — extracted from PDF
      projects: [
        { title: String, bullets: String[] }
      ]
    }
  ],
  skills: [
    { name: String, level: Number }  // level 0-100 drives animated bar width; values assigned based on professional judgement (primary langs ~85-90, secondary ~60-75)
  ],
  certifications: [
    { name: String, issuer: String, year: Number, credentialId: String }
  ],
  education: [
    { institution: String, degree: String, year: String, result: String }
  ],
  projects: [
    { subject: String, title: String, description: String, award?: String }
  ],
  activities: [
    { year: Number, items: String[] }
  ],
  referees: [
    { name: String, role: String, company: String, phone: String, email: String }
  ],
}
```

### Component Structure

```
src/
├── data/
│   └── cv.js                  # Single source of truth for all CV content
├── components/
│   ├── Navbar.jsx             # Sticky nav, auto-generates links from section list
│   ├── Hero.jsx               # Name typing effect, title, contact icons, particle bg
│   ├── About.jsx              # Personal profile paragraph
│   ├── Experience.jsx         # Animated vertical timeline
│   ├── Skills.jsx             # Animated horizontal skill bars
│   ├── Certifications.jsx     # Microsoft cert grid cards
│   ├── Education.jsx          # Education history grid
│   ├── Projects.jsx           # University project cards (2-column grid)
│   ├── Activities.jsx         # Chronological extra-curricular list
│   └── Referees.jsx           # Referee contact cards
├── hooks/
│   └── useScrollReveal.js     # Shared Framer Motion useInView + variants hook
├── App.jsx                    # Imports cv.js, passes slices to each component
├── main.jsx
└── index.css                  # Tailwind directives + CSS custom properties
```

## Animations

All animations respect the OS `prefers-reduced-motion` setting — users with that enabled get instant renders.

| Element | Animation | Implementation |
|---|---|---|
| All sections | Fade up + slide in on scroll | Framer Motion `useInView` via `useScrollReveal` hook |
| Hero name | Types character by character on load | `react-type-animation` |
| Hero background | Floating connected-dots particle network | `tsparticles` |
| Experience entries | Stagger in one by one down the timeline | Framer Motion `staggerChildren` |
| Skill bars | Fill from 0% to target width on scroll | Framer Motion `animate` on `width` |
| Navbar | Blurs + darkens background on scroll, active link highlight | Framer Motion + `scroll` event listener |
| Section headings | Underline draws left-to-right on scroll | Framer Motion SVG path animation |

### `useScrollReveal` hook

Encapsulates the `useInView` + `motion` variant pattern. Every section uses it the same way:

```js
const { ref, controls } = useScrollReveal()
// wrap section in <motion.div ref={ref} animate={controls} variants={fadeUp}>
```

## Build & Deployment

```bash
npm run dev       # Vite dev server with HMR at localhost:5173
npm run build     # Static output to dist/
npm run preview   # Preview production build locally
```

Deploy `dist/` to Vercel. Push to GitHub → connect repo in Vercel dashboard → auto-deploys on every push to main.
